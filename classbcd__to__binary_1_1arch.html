Detailed Description

Architecture definition of the BCD to binary converter.

Architecture consists of 4 processes: 1) control path : state register - Register for storing the current state of the control path logic 2) control path : next state logic - Implementation of next state logic for the control path 3) data path: data register - Register for storing the values regarding the data path, i.e. the input and output values 4) data path: routing multiplexer - Logic that changed the next value of the data path registers based on the control path state There is also combinational logic that addresses the output values and the data path status variable
